## üöß Remaining Steps for Production

### 1. Add Cloudflare Worker Secrets
Run these in WSL terminal (`npx wrangler secret put <NAME> --config wrangler.worker.jsonc`):
- `ADMIN_PASSWORD` (for `/admin/open-gate`)
- `ADMIN_USER_ID` (for `/api/auth/login`)
- `ADMIN_USER_PASSWORD` (for `/api/auth/login`)
- `RESEND_API_KEY`
- `GEMINI_API_KEY`
- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `NEXT_PUBLIC_SUPABASE_URL` (public ‚Äî set in wrangler.worker.jsonc `vars`)
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` (public ‚Äî set in wrangler.worker.jsonc `vars`)

### 2. Update Worker Name for Production
The `wrangler.worker.jsonc` currently uses `"name": "adwalididi-website-preview"`.
For the final production cutover, update to `"name": "adwalididi-website"` and update the `WORKER_SELF_REFERENCE` service binding to match.

### 3. Local Build (WSL)
```bash
sudo rm -rf node_modules   # clean up Windows symlinks
rm -rf .next               # clear Next.js build cache (fixes type errors from old deleted files)
npm install                 # install Linux binaries
npm run cf:build            # generates registries + builds worker
```

### 4. Deploy
```bash
npm run cf:deploy
# OR via GitHub Actions push to main/experimental
```
